# 语雀式嵌入在线编辑器 v8 设计说明

## GitHub 选型结论

文档编辑器建议接入 **BlockNote**：

- GitHub：<https://github.com/TypeCellOS/BlockNote>
- 定位：开源 React 块编辑器，适合知识库、文档、内嵌编辑器和协作场景。
- 关键能力：块编辑、斜杠菜单、拖拽块、缩进嵌套、格式菜单、实时协作。
- 适配原因：操作逻辑接近语雀/Notion 类知识库，适合 Markdown 快捷输入与结构化文档。

表格编辑器建议接入 **Univer**：

- GitHub：<https://github.com/dream-num/univer>
- 定位：可嵌入产品内部的 Office SDK，覆盖 Sheets、Docs、Slides。
- 关键能力：Canvas 表格渲染、公式引擎、插件化菜单、浏览器与 Node 同构处理、Facade API。
- 适配原因：比单纯表格组件更接近在线 Office，适合 xlsx 导入导出、公式、冻结、筛选、批注和协作。

备用方案：Luckysheet 已演进/改名到 Univer 方向，旧项目可作为历史参考，但新系统更建议直接采用 Univer。

## 文档页改造

页面结构：

- 左侧：知识库目录树，支持搜索、目录定位、新建文档。
- 中央：嵌入式 BlockNote 风格编辑器。
- 右侧：大纲、评论、版本、权限说明。

核心交互：

- 标题、正文、提示块、代码块、表格都作为独立 block。
- 每个 block 左侧有拖拽手柄和新增按钮。
- `/` 菜单可插入标题、任务清单、代码块、在线表格、图片/附件。
- 顶部保留导入 Markdown、导出 docx、分享、评论、成员入口。
- 文档修改进入项目变更时间线，评论锚点可回溯。

## 表格页改造

页面结构：

- 左侧：项目空间文件树，文档、表格、附件统一管理。
- 中央：嵌入式 Univer Sheets 风格表格。
- 右侧：单元格评论、版本记录、数据兼容性检查。

核心交互：

- 顶部菜单：文件、编辑、插入、格式、数据、视图、协作。
- 工具栏：撤销、重做、格式刷、字体、字号、加粗、斜体、对齐、货币、百分比、筛选、冻结、函数、图表。
- 公式栏：显示当前单元格和公式。
- 表格网格：保留行号、列头、选中单元格、协同光标、状态标签。
- 底部：工作表标签，支持新增工作表。

## 接入建议

- 文档协同：BlockNote + Yjs / Hocuspocus，后端保存 block JSON、Markdown 导出结果和版本 diff。
- 表格协同：Univer Sheets + 服务端版本存储，表格数据按 workbook 快照和增量操作记录。
- 文件兼容：导入时做预检报告，展示公式、样式、批注、图片和冻结筛选的保留率。
- 项目日志：文档 block 修改、表格单元格修改、导入导出、版本恢复都写入项目工作台右侧时间线。

## 排版优化

- 编辑页面进入专注模式：外层系统侧栏压缩为图标栏，减少和知识库目录的重复。
- 顶部全局搜索、导入、导出、新建项目在编辑态隐藏，避免干扰正文和表格操作。
- 左侧实现文件/目录定位，删除技术说明卡，仅保留用户需要的知识库结构。
- 中央编辑器面积增大，文档 block 和表格网格成为视觉主轴。
- 右侧面板只保留大纲、评论、版本、数据检查等操作相关信息。
- 技术选型名称不直接暴露给最终用户，界面文案改为“在线文档编辑器”和“在线表格编辑器”。

## 全局个性化皮肤

新增低饱和高级感皮肤，使用 CSS 变量驱动，后续可落库为用户级、团队级或项目级外观配置。皮肤是系统级能力，不隶属于文档/表格编辑器；同一套皮肤会覆盖仪表盘、项目工作台、变更时间线、在线文档、在线表格、消息同步和权限管理。

交互位置：

- 左下头像区提供“个性化”入口，用于选择系统皮肤、背景和模糊度。
- 非编辑页面顶部不再保留单独“外观”按钮，避免和项目导入、导出、新建项目混在一起。
- 文档/表格专注编辑态下，左侧系统导航压缩为图标栏，头像仍作为个性化入口，避免把皮肤能力误解为编辑器插件。
- 编辑器右侧协作面板不再放主题卡片，只保留大纲、评论、版本和数据检查，避免干扰编辑任务。

- 书信：偏暖的纸张底色、棕灰主色、轻微纸感线条，适合正式文档、会议纪要和知识沉淀。
- 情书：干净纸感底色加淡粉 UI 细节，强调柔和、私密、轻盈，但正文仍保持深色高对比。
- 风铃木：淡紫灰主色和素雅紫色边界，视觉更安静，适合长期阅读和知识库。
- 自定义：支持直接设置背景色或背景图，并根据背景自动推导 UI 主色、柔和底色、边框色和文本色；支持背景模糊度。

自定义主题建议算法：

1. 读取背景主色或背景图平均色，转换为 HSL。
2. 根据亮度选择深色或浅色文本，保证正文对比度不低于 4.5:1。
3. 主色取背景色相的低饱和版本，饱和度限制在 18%-36%，避免界面刺眼。
4. `surface` 使用半透明白或半透明深色，并叠加 `backdrop-filter` 模糊。
5. 背景模糊度建议范围 0-32px，默认 18px。

## 全局侧栏伸缩

左侧功能选择区支持矩形收缩，适用于仪表盘、项目工作台、文档、表格、消息同步和权限管理。收缩后侧栏保留图标、头像和个性化入口，隐藏项目分组、统计摘要和导航文字；主功能区同步拉伸，给甘特图、任务看板、在线编辑器和权限矩阵更大的观察与操作面积。

交互规则：

- 侧栏顶部提供收起/展开按钮，点击后侧栏宽度从完整导航过渡到图标栏。
- 文字、数字徽标、头像说明采用淡出和位移过渡，图标保持在原路径上居中，避免突然跳动。
- 主内容区域随 CSS Grid 列宽变化平滑扩展，保持页面结构稳定。
- 文档和表格编辑态默认进入图标栏模式，降低系统导航对编辑器的干扰。
- 个性化弹层在收缩态自动贴近头像展开，避免被窄侧栏裁切。
- 图标套件跟随全局皮肤变化：书信为信笺纸纹图标底，情书为柔粉圆润图标底，风铃木为淡紫花影图标底，自定义主题则使用背景模糊玻璃图标底。

## 全局排版优化

本轮优化不再围绕单一编辑器页面，而是按项目协同系统的真实工作优先级重排所有核心页面。原则是：高频决策区放大，低频参考信息压缩，说明性文字减少，日志和审计进入侧栏或紧凑列表。

- 仪表盘：压缩顶部指标卡，把主视觉空间让给任务甘特图和待处理工作队列；项目进度详情改为紧凑横向摘要，只保留风险入口和下一步动作。
- 项目工作台：缩小项目列表和变更时间线宽度，中间任务看板、任务详情和项目操作区获得更多空间；最近文档和附件降低层级，作为归档型入口保留。
- 变更时间线：保持右侧栏定位，但将筛选、搜索、日志卡片和底部导出按钮压缩，避免抢占主操作区域。
- 消息同步：消息规则配置成为主区域，发送日志收窄为辅助侧栏；渠道状态卡和模板入口压缩，重点突出规则条件和限频策略。
- 权限管理：权限矩阵占据主要宽度，审计日志和身份预览进入右侧紧凑栏，方便管理员快速扫权限而不被日志干扰。
- 全局皮肤：继续作为系统级能力保留在左下头像入口，不跟编辑器模块绑定，所有页面共享同一套低饱和视觉变量。
