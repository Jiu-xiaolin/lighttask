# LightTask v9 产品记忆

## 产品定位

LightTask 是轻量级多人项目协同系统，核心不是“在线文档合集”或“任务清单合集”，而是帮助团队把一个项目从创建、执行、协作、追踪、提醒推进到验收归档。

目标用户包括小团队、课程/教研团队、产品研发小组、运营项目组和交付项目组。系统优先服务高频项目推进场景：看风险、分任务、协作文档/表格、追变更、发提醒、做验收。

## 主业务闭环

1. 创建项目：设置目标、周期、分组、负责人和默认权限。
2. 邀请成员：项目创建者拥有完整管理权，可继续设置管理员、编辑者、评论者和只读访客。
3. 拆解任务：创建任务、依赖、起止时间、优先级、负责人和关联资料。
4. 协同执行：在项目工作台中推进任务，看板拖拽变更状态；文档、表格和附件作为项目资产被引用。
5. 留痕提醒：任务、文档、表格、附件、成员和消息变更进入项目时间线，并按规则同步飞书/微信。
6. 风险处理：仪表盘和项目工作台突出待处理动作、阻塞、逾期、待确认和消息失败。
7. 验收归档：项目完成后发起验收，冻结关键版本，生成归档包和复盘摘要。

## 不可偏移原则

1. 项目推进优先。所有模块都应服务项目生命周期，不做割裂的独立工具。
2. 行动优先于统计。仪表盘展示今天要处理什么、哪里有风险，而不是堆指标。
3. 变更可追溯。关键动作必须进入项目时间线和审计日志。
4. 协作不中断。文档和表格需要多人实时协同、评论、版本和冲突恢复，但只做到项目所需深度。
5. 主工作区优先。设置、审计、低频系统信息进入侧边栏、抽屉或管理员视图。
6. 个性化低干扰。皮肤是头像菜单里的轻能力，不占主导航和主工作区。
7. 登录入口简洁。第一版只做一个居中的账号密码登录框，框内保留一句系统简介；不做注册、营销展示和产品预览；新用户由管理员创建或邀请。

## 模块边界

### 行动仪表盘

仪表盘是“今日行动台”，核心是待处理动作、关键路径甘特、风险项目和项目进度。顶部指标只保留可行动指标，例如进行中项目、待处理动作、今日到期、待我确认。

### 项目列表与项目工作台

项目列表负责找项目、建项目、筛选风险、进入工作台。项目工作台负责推进项目，是系统的核心页面，包括任务看板/甘特、项目资产、成员、验收和右侧变更时间线。

### 项目资产

文档、表格和附件属于项目资产。可以有全局资料检索入口，但默认不把文档/表格做成与项目并列的主系统。文档和表格的修改、评论、轻量导入导出、版本回滚都要回写项目时间线。

### 消息同步

消息同步只覆盖关键协作提醒：任务到期/逾期、项目风险、评论提及、成员邀请、轻量导入导出完成、消息失败。复杂模板编排和大量自定义规则后置。

### 权限与审计

前台权限表达要简单：项目拥有者、管理员、编辑者、评论者、只读访客。后台可用 RBAC + ABAC 实现细粒度校验，但普通用户不应被复杂矩阵打扰。

### 管理中心

用户管理、权限管理、服务器监控、飞书/微信 Key 设置与状态统一进入管理中心。它是后台板块，不拆成多个普通用户主导航入口。

### 个性化

头像菜单是登录后的唯一个性化入口。点击左下角头像先显示用户卡片，卡片内承载个性签名、用户设置入口和皮肤轮播。保留书信、情书、风铃木、自定义四类皮肤；自定义优先支持背景/强调色/模糊度，复杂自动配色后置。

### 用户信息设置

用户信息设置从头像用户卡片进入，不进入主导航。第一版支持修改头像、用户卡片背景、个性签名和密码；用户卡片背景只影响个人弹出卡片，不替代全局皮肤。

## 第一版必须做

- 项目创建、成员邀请、项目内角色。
- 任务创建、看板拖拽、甘特排期、依赖与风险。
- 项目时间线、审计日志、消息提醒事件。
- 项目内轻量 Word 风格文档和轻量表格协同、评论、版本、轻量导入导出。
- 项目验收、归档包、复盘摘要。
- 仪表盘待处理队列和风险项目。

## 后置能力

- 完整 Word/WPS 级复杂编辑能力。
- 表格转储、大批量数据迁移和高保真 Office/WPS 转换。
- 复杂消息模板和规则编排。
- 单元格高级历史、复杂图表、复杂修订模式。
- 自动根据背景生成完整 UI 配色。
- 普通用户可见的系统监控、队列监控、服务拓扑。

## 当前技术记忆

- 前端建议使用 React/Next.js + TypeScript + CSS Variables/Tailwind 组织全局主题。
- 文档协同建议以 TipTap 或同类 ProseMirror 生态为底座，结合 Yjs 实时同步。
- 表格协同建议以 Univer 或同类轻量在线表格引擎为底座，结合服务端 patch、版本快照和 CSV/简单 XLSX 导入导出。
- 后端建议以模块化单体起步：项目、任务、资产、时间线、通知、权限、审计。
- 所有关键业务动作走事件中心，事件同时驱动时间线、审计和消息提醒。

## 不做的方向

- 不做营销落地页式首页，也不在登录页放产品展示侧栏。
- 不做纯装饰型玻璃效果，不牺牲可读性。
- 不把文档和表格做成脱离项目的独立产品。
- 不把个性化设计塞进主工作区。
- 不把系统监控做成普通用户主功能。
- 不为了展示功能堆满屏幕，高级功能按上下文渐进出现。
