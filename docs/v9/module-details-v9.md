# LightTask v9 模块细节

## 1. 行动仪表盘

### 页面目标

让用户在 30 秒内知道：今天要处理什么、哪些项目有风险、关键路径是否会影响交付。

### 主要内容

- 顶部行动指标：进行中项目、待处理动作、今日到期、待我确认。
- 关键路径甘特图：支持日、月、季度、年维度，展示依赖和风险。
- 今日待处理：阻塞、待确认、待审阅、消息失败、待验收。
- 项目进度详情：展示项目进度、风险数量、负责人和下一步动作。

### 交互逻辑

- 点击待处理动作直接进入项目工作台的对应任务、评论、资料或验收项。
- 点击甘特条进入任务详情；拖拽甘特条可调整排期，并自动生成时间线事件。
- 风险卡可一键提醒负责人或进入项目工作台。

### 弱化信息

纯浏览量、长期历史统计、低价值总数默认折叠或归档；指标必须能导向动作。

## 2. 项目列表与项目工作台

### 页面目标

项目列表负责找到正确项目，项目工作台负责推进项目。工作台是系统核心，不被日志、设置和低频信息占据。

### 项目列表

- 支持创建项目、搜索、筛选、归档。
- 筛选项优先保留：全部、我负责、有风险、待验收、已归档。
- 项目卡片展示名称、进度、成员、最近更新时间、风险状态和下一步动作。
- 大批量项目时支持紧凑列表。

### 项目工作台

主区按标签切换：

- 概览：目标、当前重点、关键任务、风险、最近资料、交付闭环。
- 任务：列表、看板、甘特、日历。
- 资料：项目内文档、表格、附件。
- 成员：邀请、角色、在线状态。
- 验收：验收清单、确认人、版本冻结、归档。
- 设置：项目属性、提醒规则、危险操作。

任务看板交互：

- 任务块支持拖拽到不同状态列，状态列可包含待处理、进行中、已完成、阻塞、待验收。
- 拖动过程中目标状态列高亮，释放后更新任务状态，并自动写入右侧变更时间线。
- 如果用户无权限、依赖未完成或目标状态不允许流转，目标列显示不可投放状态并提示原因。
- 状态变更触发权限校验、任务变更事件、审计记录和消息规则匹配。

右侧固定变更时间线：

- 任务变更、资料版本、附件操作、成员权限、消息提醒和验收归档均以卡片记录。
- 卡片颜色按类型区分，但必须配合图标和标题，不只依赖颜色。
- 时间线支持类型、成员、时间范围筛选。
- 重要日志可置顶到项目概览。

### 项目验收与归档

- 负责人可发起验收。
- 验收清单应关联关键任务、文档、表格和附件。
- 验收通过后冻结关键资料版本。
- 归档包包含任务清单、资料快照、时间线摘要和复盘摘要。

## 3. 项目资料

### 页面目标

在项目上下文中管理文档、表格和附件。资料不是独立产品，而是项目推进过程中的证据、协作内容和交付物。

### 文档能力

第一版重点：

- 标题、列表、待办、表格、图片、附件、代码块。
- Markdown 快捷输入和斜杠菜单。
- 评论线程、多人光标、实时保存。
- 自动版本、版本对比、回滚。
- 导入 markdown / html / 简单 docx，导出 markdown / html / 浏览器打印 pdf。
- 文档更新、评论、回滚写入项目时间线。

后置能力：

- 复杂分页、分栏、完整修订模式、复杂打印样式。
- 高保真 Word 排版兼容和服务端 Office 转换。

### 表格能力

第一版重点：

- 单元格编辑、公式栏、基础函数。
- 字体、对齐、边框、填充、数字格式。
- 冻结、筛选、排序。
- 评论、多工作表、多人选区。
- 自动快照、回滚。
- 导入 csv / 简单 xlsx，导出 csv / 简单 xlsx。
- 导入后只生成轻量兼容提示。

后置能力：

- 表格转储、大批量数据迁移、复杂图表、数据透视表、宏、极细粒度单元格历史。

### 资料与任务的关系

- 任务可关联多个资料。
- 评论可转为任务。
- 资料轻量导入导出完成后进入项目时间线。
- 资料被验收引用后，归档时冻结对应版本。

## 4. 消息同步

### 页面目标

把项目关键动作推送给团队，减少用户频繁进入系统检查的成本。

### 第一版渠道

- 飞书机器人。
- 微信公众号。

### 第一版规则

- 任务到期。
- 任务逾期。
- 项目风险。
- 评论提及。
- 成员邀请。
- 轻量导入导出完成。
- 消息发送失败。

### 操作

- 新建规则。
- 测试发送。
- 启停规则。
- 查看发送日志。
- 失败重试。

### 后置能力

复杂模板管理、静默时段、复杂条件编排和多渠道细分策略后置。

## 5. 登录与管理中心

### 登录页

登录页是系统入口，不承担营销介绍、产品预览和注册流程。视觉上保持优雅、安静、高端、低干扰，只保留一个居中的登录框。

登录框内允许保留极简简介，用一句话说明系统定位，例如“轻量项目协同系统，用来推进任务、资料、消息和验收归档”。简介必须在登录框内部，不在页面侧边额外做展示区。

第一版只提供：

- 账号登录。
- 密码登录。
- 保持登录。
- 忘记密码入口。
- 登录后进入行动仪表盘或最近访问项目。

不做：

- 开放注册。
- 营销落地页式介绍。
- 产品展示侧栏、宣传文案和装饰预览。
- 第三方登录。
- 复杂组织申请流程。

新用户由超级管理员在管理中心创建或邀请。

### 登录后账户卡片与个人设置

该能力只在登录后的应用框架内出现，和登录框简介是两个需求。头像固定保留在左下角账户区；侧边栏收起后头像仍以图标按钮形式存在。点击头像弹出用户卡片，卡片内显示头像、姓名、角色、个性签名和个性化皮肤卡片。

交互路径：

- 第一次点击头像：展开用户卡片和皮肤轮播。
- 再次点击用户卡片：进入用户信息设置页。
- 用户信息设置支持修改头像、用户卡片背景、个性签名和密码。
- 用户卡片背景只影响个人卡片；全局界面皮肤仍通过皮肤卡片切换。

### 管理中心

### 页面目标

把用户、权限、服务器监控、飞书/微信 Key 设置与状态合并到一个后台板块。普通用户不进入该页；项目管理员只能看到与自己项目有关的成员和权限。

### 子板块

- 用户管理：创建用户、邀请用户、启用/禁用、在线状态、最近登录。
- 权限管理：角色模板、权限范围、邀请链接和审计日志。
- 服务器监控：CPU、内存、磁盘、WebSocket 在线连接、PostgreSQL/Redis、Worker 队列、低配部署健康度。
- 通知 Key：飞书机器人 Key、微信公众号 Key、加密保存、状态检测、测试发送。

### 角色

- 超级管理员：查看全部数据，管理系统配置、全局用户、消息渠道和审计。
- 普通用户：可创建项目，可参与被邀请项目。
- 项目拥有者：项目创建者，拥有该项目完整管理权。
- 项目管理员：管理成员、任务、资料和项目配置。
- 编辑者：编辑任务、文档、表格和附件。
- 评论者：评论任务和资料。
- 只读访客：查看被授权内容。

### 权限维度

- 系统级权限。
- 项目级权限。
- 任务级权限。
- 资料级权限。
- 消息渠道级权限。

### 审计

权限变更、邀请、移除成员、分享链接、版本回滚、导出数据、项目归档等操作必须进入审计日志。

### 低配部署监控

2 核 2G 40G 极简安装模式下，管理中心需要突出 CPU、内存、磁盘和 Worker 队列，避免系统在低配环境中因为日志、上传文件或后台任务失控。

### Key 安全

飞书和微信公众号 Key 只能加密保存，前端只显示脱敏状态、最近检测时间和测试发送结果，不显示明文。

## 6. 个性化

### 入口

个性化入口只保留在登录后的头像菜单用户卡片中。点击头像后先弹出用户卡片，再在卡片下方显示皮肤轮播，不再提供其他独立切换页面或主区皮肤面板。

### 第一版能力

- 书信、情书、风铃木三套预设。
- 自定义背景、强调色和模糊度。
- 选中后全局应用，包括导航、按钮、面板、图表和编辑器。

### 后置能力

- 复杂背景识别和全自动 UI 配色。
- 每套皮肤独立图标风格。
