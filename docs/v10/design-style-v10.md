# LightTask v10 设计风格规范

## 风格关键词

优雅、清爽、低饱和、高留白、轻玻璃、纸感、专业工具、顺滑动效、低干扰协同。

视觉方向不是强烈的拟物，也不是营销式玻璃炫技，而是接近高级生产力工具：界面克制，内容清楚，操作可预期，动效服务空间关系。所有视觉决策都服务项目推进和成员进度判断，不让装饰、个性化或系统信息抢占主工作区。

## 全局布局

### 桌面端基础结构

- 左侧导航：展开 220-248px，收起 56-68px。
- 顶部栏：48-56px，放搜索、关键操作、协作状态和头像，不堆满按钮。
- 主工作区：占页面最大比例，根据当前模块动态分配。
- 右侧边栏：用于时间线、评论、大纲、属性，不与主工作区抢焦点。

### 页面重点比例

- 登录页：不带主导航和顶部栏；只保留一个居中的登录卡片，卡片内可有一句系统简介；不放产品状态预览、侧边展示区、营销文案和注册入口。
- 行动仪表盘：关键路径甘特图占 42%-50% 主视觉区域；我的进度与快慢天数占 18%-24%；今日待处理占 20%-26%；项目进度详情占 14%-20%。
- 项目工作台：任务/成员进度/提交物主区占 65%-75%；右侧变更时间线宽 280-340px；设置、导出和低频信息折叠到抽屉。
- 项目文件：文件树与编辑器占主要区域；左侧文件树可收起；Word、表格、附件点一下在同一工作区打开；右侧文件属性、大纲、评论、版本按需切换。
- 消息：关键规则和失败日志优先，复杂模板详情进入右侧抽屉。
- 管理：用户管理和权限管理在主区，服务器监控和飞书/微信 Key 状态放右侧，复杂权限矩阵默认折叠到高级配置。

## 信息层级

1. 当前要做的动作和要看的内容最大。
2. 风险、逾期、待确认动作必须清晰但不刺眼。
3. 统计数字只保留可行动指标，个人快慢天数、预计完成日期和文件提交状态优先于纯展示数字。
4. 历史日志、审计、系统细节进入侧边栏、抽屉或筛选视图。
5. 文档和表格作为项目文件中的文件类型出现，减少独立产品感；文件树、编辑框体、工具栏、属性栏要成为一个整体。
6. 甘特图默认展示项目主线，成员节点默认聚合；通过筛选查看指定成员，避免过载。

## 主题系统

### 书信

- 气质：温润、纸张、安静、适合长期阅读。
- 背景：浅米白纸感纹理。
- 主色：墨蓝、暖灰。
- 强调色：低饱和金棕。
- 图标：细线、略带手写纸面气质。

### 情书

- 气质：干净纸感底色上带一点淡粉。
- 背景：浅粉白、极轻柔纸纹。
- 主色：深灰紫。
- 强调色：玫瑰粉、雾粉。
- 图标：圆润、柔和、线条稍轻。

### 风铃木

- 气质：素雅紫、轻盈、清透。
- 背景：浅紫灰与低饱和花影。
- 主色：紫灰、深蓝灰。
- 强调色：风铃木淡紫。
- 图标：细线、轻微弧度、留白更大。

### 自定义

- 第一版支持本地图片或纯色背景、背景模糊度和手动强调色。
- 背景复杂时自动加半透明遮罩保证文本对比度。
- 自动推导完整 UI 配色后置。

## 组件规则

- 卡片圆角不超过 8px，编辑器纸张可以使用 6-8px。
- 图标按钮最小点击区域 36px，移动端不少于 44px。
- 工具栏按钮全部提供 tooltip 和 aria-label。
- 不用 emoji 做功能图标。
- 主按钮只用于创建、保存、分享、导入、确认等明确命令。
- 文档和表格工具栏用图标为主，文字只出现在下拉、tooltip、状态栏和必要标签；文件管理的新增、上传、重命名、删除、导出必须清晰可见。
- 成员进度卡需要清晰显示成员、状态、快慢天数和提交物状态，不展示工时和绩效排名。
- 任务上报按钮使用清晰动作词：完成、延期、阻塞、放弃、上传成果、休息一下、继续任务。
- 文件收集箱按任务和成员分组，默认展示未交、需修改和待验收文件。
- 不做卡片套卡片。页面区块应是自然分区，不做过多浮层堆叠。
- 系统监控、复杂权限、复杂消息规则不进入普通用户主导航。
- 登录后头像固定在左下角账户区；侧边栏收起时也保留头像图标，不能被隐藏或挤出视口。
- 用户卡片从头像位置弹出，负责承载用户信息、个性签名、用户设置入口、皮肤轮播和退出登录；退出登录固定在二级菜单底部，使用低干扰危险色，不放入主导航。

## 动效规范

- 常规状态切换：180-240ms。
- 侧边栏收起、头像用户卡片弹出、皮肤卡片切换：260-340ms。
- 曲线建议：cubic-bezier(0.2, 0.8, 0.2, 1)。
- 动效需要表达空间关系，例如侧栏收起后主区扩展、皮肤卡片从头像位置生长。
- 避免装饰性循环动画。
- 支持 reduced motion，关闭非必要位移与模糊动画。

## 可访问性与可用性

- 正文对比度不低于 4.5:1。
- 图标按钮必须可键盘聚焦。
- 色彩不能作为唯一状态表达，风险状态同时使用图标、标签或文字。
- 工具栏密集区域要保证焦点顺序与视觉顺序一致。
- 导入导出、版本回滚、权限变更等高风险操作需要明确二次确认。
