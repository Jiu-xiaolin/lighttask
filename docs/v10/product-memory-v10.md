# LightTask v10 产品记忆

## 产品定位

LightTask 是轻量级多人项目协同系统，核心不是“在线文档合集”或“任务清单合集”，而是帮助团队把一个项目从创建、执行、协作、追踪、提醒推进到验收归档。v10 在 v9 基础上补齐成员级进度、计划对比、个人进度和任务提交物，让项目创建者和成员都能清楚看到“谁完成了什么、比计划快慢多少、文件是否已交”。

目标用户包括小团队、课程/教研团队、产品研发小组、运营项目组和交付项目组。系统优先服务高频项目推进场景：看风险、分任务、协作文档/表格、追变更、发提醒、做验收。

## 主业务闭环

1. 创建项目：设置目标、周期、分组、负责人和默认权限。
2. 邀请成员：项目创建者拥有完整管理权，可继续设置管理员、编辑者、评论者和只读访客。
3. 拆解任务：创建任务、依赖、起止时间、优先级、负责人和关联资料。
4. 分配成员进度：一个任务可分配给一个或多个成员，每个成员有计划、实际、状态、说明和提交物。
5. 协同执行：在项目工作台中推进任务，看板拖拽变更状态；成员可上报完成、延期、阻塞、放弃、休息或继续任务；Word、表格和附件作为项目文件被引用。
6. 留痕提醒：任务、成员进度、项目文件、附件、提交物、成员和消息变更进入项目时间线，并按规则同步飞书/微信。
7. 风险处理：仪表盘和项目工作台突出待处理动作、个人延期、项目阻塞、逾期、待确认和消息失败。
8. 验收归档：项目完成后发起验收，冻结关键版本，统计成员完成时间、快慢天数和文件提交状态，生成归档包和复盘摘要。

## 不可偏移原则

1. 项目推进优先。所有模块都应服务项目生命周期，不做割裂的独立工具。
2. 行动优先于统计。仪表盘展示今天要处理什么、哪里有风险，而不是堆指标。
3. 变更可追溯。关键动作必须进入项目时间线和审计日志。
4. 协作不中断。文档和表格需要多人实时协同、评论、版本和冲突恢复，但只做到项目所需深度。
5. 成员进度必须轻量。v10 只追踪任务推进所需的计划、实际、状态和提交物，不做复杂工时、考勤或绩效系统。
6. 主工作区优先。设置、审计、低频系统信息进入侧边栏、抽屉或管理员视图。
7. 个性化低干扰。皮肤是头像菜单里的轻能力，不占主导航和主工作区。
8. 登录入口简洁。第一版只做一个居中的账号密码登录框，框内保留一句系统简介；不做注册、营销展示和产品预览；新用户由管理员创建或邀请。

## 模块边界

### 行动仪表盘

仪表盘是“今日行动台 + 我的进度台”，核心是待处理动作、关键路径甘特、风险项目、项目进度和个人进度。顶部指标只保留可行动指标，例如进行中项目、待处理动作、今日到期、待我确认、我的完成率、预计完成日期和快慢天数。

### 项目列表与项目工作台

项目列表负责找项目、建项目、筛选风险、进入工作台。项目工作台负责推进项目，是系统的核心页面，包括任务看板/甘特、成员级进度、任务上报、任务提交物、项目资产、成员、验收和右侧变更时间线。

### 成员级任务进度

成员级进度是 v10 的核心补充。一个任务可有多个成员进度项，每个进度项记录成员、计划开始/结束、当前计划结束、实际完成时间、状态、快慢天数、上报说明和关联提交物。它支撑学习项目的个人统计，也支撑多人项目的成员完成节点。

### 计划对比

任务和成员进度均保留三层时间：原计划、当前计划、实际进度。原计划作为基线不轻易覆盖；当前计划可因延期调整；实际进度来自成员开始、完成、休息、继续任务和提交文件等节点。系统用这三层时间计算预计完成日期和快慢天数。

### 项目文件

Word 文档、在线表格、附件和提交物属于项目文件。可以有全局文件检索入口，但默认不把文档/表格做成与项目并列的主系统。项目文件提供统一入口，支持新建、上传、重命名、删除、恢复、移动、导入、导出；点击 Word 文件打开嵌入式文档编辑器，点击表格文件打开嵌入式表格编辑器，点击附件进入预览或下载。文档和表格的修改、评论、轻量导入导出、版本回滚都要回写项目时间线。v10 新增任务提交物和项目文件收集箱，成员可把本地 Word、表格、图片或其他成果提交到自己的任务进度中，项目创建者按任务和成员统一收集。

### 消息同步

消息同步只覆盖关键协作提醒：任务下发、任务到期/逾期、成员完成、成员延期、成员放弃、项目风险、评论提及、成员邀请、轻量导入导出完成、文件提交、项目全部完成、消息失败。复杂模板编排和大量自定义规则后置。

### 权限与审计

前台权限表达要简单：项目拥有者、管理员、编辑者、评论者、只读访客。后台可用 RBAC + ABAC 实现细粒度校验，但普通用户不应被复杂矩阵打扰。v10 需要把“进度可见”和“文件可见”拆开：成员可以互看进度甘特，但是否能查看或下载别人的提交文件由项目权限控制。

### 管理中心

用户管理、权限管理、服务器监控、飞书/微信 Key 设置与状态统一进入管理中心。它是后台板块，不拆成多个普通用户主导航入口。

### 个性化

头像菜单是登录后的唯一个性化入口，也承载退出登录。点击左下角头像先显示用户卡片，卡片内承载个性签名、用户设置入口、皮肤轮播和退出登录；退出登录固定在二级菜单底部，不进入主导航、顶部栏或管理中心。保留书信、情书、风铃木、自定义四类皮肤；自定义优先支持背景/强调色/模糊度，复杂自动配色后置。

### 用户信息设置

用户信息设置从头像用户卡片进入，不进入主导航。第一版支持修改头像、用户卡片背景、个性签名和密码；用户卡片背景只影响个人弹出卡片，不替代全局皮肤。

## 第一版必须做

- 项目创建、成员邀请、项目内角色。
- 任务创建、看板拖拽、甘特排期、依赖与风险。
- 成员级任务进度、计划基线、当前计划、实际完成节点和快慢天数。
- 个人进度仪表盘，展示今日、本周、本月完成情况和预计完成日期。
- 任务上报动作：完成、延期、阻塞、放弃、休息一下、继续下一任务、上传成果、提醒创建者。
- 项目文件统一管理，包含 Word、表格、附件、任务文件提交和项目文件收集箱。
- 项目时间线、审计日志、消息提醒事件。
- 项目文件内轻量 Word 风格文档和轻量表格协同、评论、版本、轻量导入导出。
- 项目验收、成员完成统计、归档包、复盘摘要。
- 仪表盘待处理队列和风险项目。

## 后置能力

- 完整 Word/WPS 级复杂编辑能力。
- 表格转储、大批量数据迁移和高保真 Office/WPS 转换。
- 复杂消息模板和规则编排。
- 单元格高级历史、复杂图表、复杂修订模式。
- 自动根据背景生成完整 UI 配色。
- 普通用户可见的系统监控、队列监控、服务拓扑。
- 复杂学习系统、考勤系统、工时系统和绩效系统。

## 当前技术记忆

- 前端建议使用 React/Next.js + TypeScript + CSS Variables/Tailwind 组织全局主题。
- 文档协同建议以 TipTap 或同类 ProseMirror 生态为底座，结合 Yjs 实时同步。
- 表格协同建议以 Univer 或同类轻量在线表格引擎为底座，结合服务端 patch、版本快照和 CSV/简单 XLSX 导入导出。
- 后端建议以模块化单体起步：项目、任务、项目文件、时间线、通知、权限、审计。
- 所有关键业务动作走事件中心，事件同时驱动时间线、审计和消息提醒。
- v10 新增 `task_assignments`、`task_progress_reports`、`task_submissions`、`task_plan_snapshots` 等核心数据概念。

## 不做的方向

- 不做营销落地页式首页，也不在登录页放产品展示侧栏。
- 不做纯装饰型玻璃效果，不牺牲可读性。
- 不把文档和表格做成脱离项目的独立产品。
- 不把个性化设计塞进主工作区。
- 不把系统监控做成普通用户主功能。
- 不做重型网盘、完整学习平台、工时考勤和绩效排名。
- 不为了展示功能堆满屏幕，高级功能按上下文渐进出现。
