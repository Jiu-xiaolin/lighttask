# 轻协同前端 UI v4 优化说明

日期：2026-06-01  
依据：ui-ux-pro-max 的 SaaS / Dashboard / Admin Panel 设计原则

## 1. 本轮优化目标

v3 已完成基础视觉方向，但仍偏“展示型后台”。v4 重点改为“可处理的协同工作台”，让用户在看到风险、任务、文档变更、提醒失败和权限变化后，可以直接进入下一步动作。

## 2. 仪表盘优化

新增：

- 待处理工作队列：阻塞任务、失败提醒、导入兼容确认都转成可点击动作。
- 甘特冲突提示：展示任务依赖、负责人负载和推荐调整。
- 项目进度卡：每个项目进度都带下一步按钮，例如处理风险、查看文档、查看规则。

设计收益：

- 仪表盘从“看数据”升级为“处理今日关键事项”。
- 风险项不再只用颜色表达，配有明确原因和行动按钮。

## 3. 项目工作台优化

新增：

- 选中任务详情卡：展示负责人、截止时间、依赖、完成度和快捷操作。
- 时间线 diff：任务进度、成员权限、文档版本和附件变更展示前后变化。
- 附件与在线文档仍保留在同一项目上下文中，减少跳转。

设计收益：

- 用户点击任务后能直接判断当前状态和下一步。
- 时间线更像审计日志，能支持后续做回滚、差异预览和责任追踪。

## 4. 在线文档与表格优化

新增：

- 多人协作状态条：实时同步、评论锚点、格式兼容待确认。
- 正在输入状态：显示具体协作成员正在编辑。
- 评论锚点：正文区域直接标出评论数量。
- 版本差异：版本卡片展示 3 项到 4 项的变化。
- Office / WPS 导入预检：显示样式、批注等兼容率。

设计收益：

- 更接近石墨文档类多人协作体验。
- 导入导出不再只是按钮，而是可预检、可确认、可追踪的流程。

## 5. 消息同步优化

新增：

- 规则模板：到期提醒、阻塞升级、文档评论、成员变更。
- 防打扰限制：同一任务 2 小时内最多提醒 1 次。
- 发送日志保留成功、失败、重试状态。

设计收益：

- 降低消息轰炸风险。
- 配置流程更容易复用，不需要每条规则从零设置。

## 6. 权限管理优化

新增：

- 以用户身份预览最终权限。
- 审计日志与项目级管理权说明放在同一侧栏。

设计收益：

- 解决“角色权限 + 项目级覆写”叠加后不直观的问题。
- 超级管理员和项目创建者的边界更清楚。
