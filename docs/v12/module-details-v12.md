# LightTask v12 模块细节

## 1. 行动仪表盘

### 页面目标

让用户在 30 秒内知道：今天要处理什么、哪些项目有风险、关键路径是否会影响交付，以及自己当前学习/项目进度比计划快了还是慢了。

### 主要内容

- 顶部行动指标：今日待处理、我的进度偏差、待收集文件、风险项目。
- 顶部个人指标合并进“我的进度”卡：今日、本周、本月、预计完成日期、比计划快/慢天数。
- 关键路径甘特图：支持日、月、季度、年维度，展示依赖、风险、原计划、当前计划和实际进度。
- 甘特默认筛选：项目主线、我负责、只看延期、成员。
- 甘特高级筛选：原计划对比、当前进度、团队完成节点进入“视图设置”。
- 今日待处理：阻塞、延期上报、待确认、待审阅、消息失败、待验收。
- 项目进度详情：展示项目进度、风险数量和下一步动作；负责人和历史明细默认折叠。

### 交互逻辑

- 点击待处理动作直接进入项目工作台的对应任务、评论、文件或验收项。
- 点击甘特条进入任务详情；拖拽甘特条可调整排期，并自动生成时间线事件。
- 风险卡可一键提醒负责人或进入项目工作台。
- 点击“我的进度”进入个人任务清单，可按日、周、月查看完成情况和预计完成日期。
- 切换成员甘特时只展示该成员的任务节点，避免团队甘特过载。
- “只看延期”和“只看我负责”属于快速过滤，不改变项目数据，只改变当前视图。
- 团队完成节点默认以头像/数字聚合，点击任务条后再展开到具体成员节点。
- 快慢天数按原计划结束时间和实际/预计完成时间计算，负数表示提前，正数表示延期。

### 弱化信息

纯浏览量、长期历史统计、低价值总数默认折叠或归档；指标必须能导向动作。预计完成日期不再单独占顶部指标，统一放入我的进度卡。个人进度只展示任务推进所需信息，不做考勤、工时和绩效排名。

## 2. 项目列表与项目工作台

### 页面目标

项目列表负责找到正确项目，项目工作台负责推进项目。工作台是系统核心，不被日志、设置和低频信息占据。

### 项目列表

- 支持创建项目、搜索、筛选、归档。
- 支持项目查看、编辑、删除/关闭、归档、恢复归档和项目设置修改。
- 筛选项优先保留：全部、我负责、有风险、待验收、已归档。
- 项目卡片展示名称、进度、成员、最近更新时间、风险状态和下一步动作。
- 大批量项目时支持紧凑列表。

### 项目工作台

主区按标签切换：

- 概览：目标、当前重点、关键任务、风险、最近文件、交付闭环。
- 任务：列表、看板、甘特、日历、成员进度、任务上报。
- 文件：项目内 Word、表格、附件、任务提交物、文件收集箱。
- 成员：邀请、角色、在线状态。
- 验收：验收清单、确认人、版本冻结、归档。
- 设置：项目属性、提醒规则、危险操作。

v12 首屏默认只常驻当前重点、任务看板和右侧变更时间线。成员进度、文件收集箱、任务上报和验收统计以摘要入口展示，进入对应标签后再展开详情。

任务看板交互：

- 任务支持查看、创建、编辑、删除、复制、归档和恢复。
- 任务编辑应覆盖标题、描述、负责人、成员进度、起止时间、优先级、依赖任务和关联资料。
- 任务块支持拖拽到不同状态列，状态列可包含待处理、进行中、已完成、阻塞、待验收。
- 拖动过程中目标状态列高亮，释放后更新任务状态，并自动写入右侧变更时间线。
- 如果用户无权限、依赖未完成或目标状态不允许流转，目标列显示不可投放状态并提示原因。
- 状态变更触发权限校验、任务变更事件、审计记录和消息规则匹配。

成员级进度：

- 一个任务可以有多个成员进度项。
- 成员进度支持新增、查看详情、编辑计划、移除进度和重新分配负责人。
- 每个进度项记录成员、原计划、当前计划、实际完成、状态、快慢天数、说明和提交物。
- 学习项目中，成员进度可以表示章节、课程、练习和复盘的个人完成情况。
- 多人项目中，成员进度可以表示某个共同任务下每个人负责的部分。
- 成员可以互看进度甘特；是否能看对方提交文件由文件权限控制。

任务上报动作：

- 完成：写入实际完成时间，并在甘特图标记成员完成节点。
- 延期：必须选择延期到哪一天并填写原因，更新当前计划，写入任务备注和时间线。
- 阻塞：填写阻塞原因，可提醒项目创建者或指定成员。
- 放弃：必须填写原因，进入风险队列并提醒创建者。
- 上传成果：把 Word、表格、图片或附件提交到当前成员进度。
- 休息一下：标记阶段完成但不立即开始下一任务。
- 继续下一任务：生成个人新节点，进入下一任务或下一阶段。
- 提醒创建者：向项目创建者发送飞书/微信或系统内提醒。

甘特图复杂度控制：

- 默认显示项目主线，不直接铺开全部成员节点。
- 首层只显示项目主线、我负责、延期和成员；原计划、当前进度、实际节点、团队汇总和团队完成节点进入视图设置。
- 成员完成、延期、休息、继续任务都以轻节点显示，默认聚合，点击后展开。
- 延期节点在任务备注和右侧时间线同步展示。

任务提交物与文件收集箱：

- 成员在任务进度中上传成果文件，形成任务提交物。
- 任务提交物支持上传、查看、重新提交、撤回/删除、标记需修改和标记已验收。
- 项目创建者在文件收集箱按任务、成员、状态统一查看。
- 提交状态包括未交、已交、需修改、已验收。
- 文件可关联验收清单，验收通过后冻结对应版本。
- 成员能否查看别人文件由文件可见、文件下载权限控制。

右侧固定变更时间线：

- 任务变更、文件版本、附件操作、成员权限、消息提醒和验收归档均以卡片记录。
- 卡片颜色按类型区分，但必须配合图标和标题，不只依赖颜色。
- 时间线支持类型、成员、时间范围筛选。
- 重要日志可置顶到项目概览。

### 项目验收与归档

- 负责人可发起验收。
- 验收清单应关联关键任务、文档、表格和附件。
- 验收项支持新增、查看、编辑、删除、关联任务/文件/提交物。
- 验收项状态支持通过、需修改、驳回和重新提交。
- 验收前检查每个成员提交物是否齐全。
- 验收统计报告统计每个成员计划完成时间、实际完成时间、快/慢天数、延期次数、放弃次数、阻塞次数、提交文件数量和是否验收通过。
- 验收报告在项目工作台默认只展示精简摘要；完整明细进入验收页或归档包。
- 验收报告生成、通过、驳回和归档必须写入右侧时间线。
- 验收通过后冻结关键文件版本。
- 归档包包含任务清单、文件快照、时间线摘要和复盘摘要。

## 3. 项目文件

### 页面目标

在项目上下文中统一管理 Word 文档、在线表格、附件和任务提交物。项目文件不是独立网盘，而是项目推进过程中的证据、协作内容和交付物；用户点一下文件即可在同一工作区打开对应编辑器或预览。

### 文件管理能力

第一版重点：

- 左侧文件树统一展示 Word、表格、附件、任务提交物和文件收集箱。
- 支持新建 Word 文档、新建在线表格、上传本地文件。
- 支持查看、重命名、删除、恢复、移动目录、导入和导出。
- Word 文件打开嵌入式文档编辑器；表格文件打开嵌入式表格编辑器；附件打开预览或下载。
- 文件属性、版本、评论、大纲、提交物状态放在右侧辅助栏标签中；默认只打开大纲或评论。
- 文件创建、重命名、删除、上传、导入导出、版本回滚写入项目时间线和审计日志。

后置能力：

- 重型网盘、复杂文件夹权限继承、全文 OCR、超大文件预览和批量文件自动分类。

### 文档能力

第一版重点：

- 标题、列表、待办、表格、图片、附件、代码块。
- Markdown 快捷输入和斜杠菜单。
- 评论线程、多人光标、实时保存。
- 自动版本、版本对比、回滚。
- 导入 markdown / html / 简单 docx，导出 markdown / html / 浏览器打印 pdf。
- 文档更新、评论、回滚写入项目时间线。

后置能力：

- 复杂分页、分栏、完整修订模式、复杂打印样式。
- 高保真 Word 排版兼容和服务端 Office 转换。

### 表格能力

第一版重点：

- 单元格编辑、公式栏、基础函数。
- 字体、对齐、边框、填充、数字格式。
- 冻结、筛选、排序。
- 评论、多工作表、多人选区。
- 自动快照、回滚。
- 导入 csv / 简单 xlsx，导出 csv / 简单 xlsx。
- 导入后只生成轻量兼容提示。

后置能力：

- 表格转储、大批量数据迁移、复杂图表、数据透视表、宏、极细粒度单元格历史。

### 文件与任务的关系

- 任务可关联多个文件。
- 成员进度可关联自己的提交物。
- 文件收集箱按任务、成员和提交状态聚合项目文件。
- 评论可转为任务。
- 文件轻量导入导出完成后进入项目时间线。
- 文件被验收引用后，归档时冻结对应版本。

## 4. 消息同步

### 页面目标

把项目关键动作推送给团队，减少用户频繁进入系统检查的成本。

### 第一版渠道

- 飞书机器人。
- 微信公众号。

### 第一版规则

- 任务到期。
- 任务逾期。
- 任务下发。
- 成员完成。
- 成员延期。
- 成员放弃。
- 文件提交。
- 项目风险。
- 评论提及。
- 成员邀请。
- 轻量导入导出完成。
- 项目全部完成。
- 消息发送失败。

### 操作

- 新建规则。
- 查看规则详情。
- 编辑规则。
- 删除规则。
- 复制规则。
- 测试发送。
- 启停规则。
- 查看发送日志。
- 失败重试。
- 选择提醒对象：全部成员、负责人、项目创建者或自定义成员。
- 选择“不提醒”：允许只写系统时间线和待处理队列，不推送飞书/微信。
- 自定义成员：从项目成员中选择，可组合负责人、创建者和具体成员。
- 任务下发、延期、完成、文件未交、验收通过默认带推荐提醒对象，但用户可以修改。
- v12 默认不展开定向提醒选择器；只有新建规则、编辑规则或点击规则行时展开。

### 后置能力

复杂模板管理、静默时段、复杂条件编排和多渠道细分策略后置。

## 5. 登录与管理中心

### 登录页

登录页是系统入口，不承担营销介绍、产品预览和注册流程。视觉上保持优雅、安静、高端、低干扰，只保留一个居中的登录框。

登录框内允许保留极简简介，用一句话说明系统定位，例如“轻量项目协同系统，用来推进任务、资料、消息和验收归档”。简介必须在登录框内部，不在页面侧边额外做展示区。

第一版只提供：

- 账号登录。
- 密码登录。
- 保持登录。
- 忘记密码入口。
- 登录后进入行动仪表盘或最近访问项目。

登录安全规则：

- 如果用户开启 IP 白名单，只有来源 IP 命中白名单时才允许登录。
- 未命中白名单时不提示具体白名单内容，只提示当前网络不允许访问，并写入安全审计。
- 登录状态在连续未操作 24 小时后销毁登录凭证，用户再次访问需要重新登录。

不做：

- 开放注册。
- 营销落地页式介绍。
- 产品展示侧栏、宣传文案和装饰预览。
- 第三方登录。
- 复杂组织申请流程。

新用户由超级管理员在管理中心创建或邀请。

### 登录后账户卡片与个人设置

该能力只在登录后的应用框架内出现，和登录框简介是两个需求。头像固定保留在左下角账户区；侧边栏收起后头像仍以图标按钮形式存在。点击头像弹出用户卡片，卡片内显示头像、姓名、角色、个性签名和个性化皮肤卡片。

交互路径：

- 第一次点击头像：展开用户卡片和皮肤轮播。
- 再次点击用户卡片：进入用户信息设置页。
- 退出登录：放在头像弹出的二级菜单底部，使用低干扰危险操作样式，点击后结束当前会话并回到登录框。
- 用户信息设置支持修改头像、用户卡片背景、个性签名和密码。
- 用户卡片背景只影响个人卡片；全局界面皮肤仍通过皮肤卡片切换。

### 管理中心

### 页面目标

把用户、权限、服务器监控、飞书/微信 Key 设置与状态合并到一个后台板块。普通用户不进入该页；项目管理员只能看到与自己项目有关的成员和权限。

### 子板块

- 用户管理：创建用户、邀请用户、查看用户详情、编辑用户、启用/禁用、重置密码、用户级 IP 白名单、登录会话查看与吊销、删除/停用用户、在线状态、最近登录。
- 权限管理：角色模板增删改查、权限范围查看/修改、邀请链接和审计日志。
- 服务器监控：默认只显示健康摘要；点击后查看 CPU、内存、磁盘、WebSocket、PostgreSQL/Redis、Worker 队列。
- 通知 Key：默认只显示飞书/微信状态摘要；点击后查看新增渠道、修改 Key、启用/禁用、删除渠道配置、状态检测和测试发送。

### 角色

- 超级管理员：查看全部数据，管理系统配置、全局用户、消息渠道和审计。
- 普通用户：可创建项目，可参与被邀请项目。
- 项目拥有者：项目创建者，拥有该项目完整管理权。
- 项目管理员：管理成员、任务、文件和项目配置。
- 编辑者：编辑任务、文档、表格和附件。
- 评论者：评论任务和文件。
- 只读访客：查看被授权内容。

### 权限维度

- 系统级权限。
- 项目级权限。
- 任务级权限。
- 文件级权限。
- 进度可见权限：能否查看项目成员甘特、成员状态和快慢天数。
- 文件可见权限：能否查看别人上传的任务提交物和项目文件。
- 文件下载权限：能否下载或导出授权文件。
- 提交物验收权限：能否标记已验收、需修改或驳回。
- 消息渠道级权限。

### 凭证与操作绑定

用户所有操作都必须绑定当前有效凭证。后端从会话凭证解析用户身份，再按用户状态、系统角色、项目成员关系、任务负责人、文件授权和权限范围逐层判断。

判断顺序：

1. 凭证是否有效，用户是否启用。
2. 凭证是否连续未操作超过 24 小时。
3. 用户开启 IP 白名单时，当前请求来源 IP 是否命中白名单。
4. 用户是否具备系统级入口权限。
5. 用户是否属于目标项目，或是否为超级管理员。
6. 用户是否拥有该资源的查看、编辑、删除、下载、导出、验收或管理权限。
7. 资源当前状态是否允许该操作，例如已归档项目、冻结验收版本、被删除文件不能继续编辑。

权限不足时：

- 后端直接拒绝操作，不创建任务、不修改数据、不发消息、不生成下载链接。
- 前端按钮需要隐藏或禁用，并展示简短原因。
- 敏感失败行为进入安全审计，例如越权下载、越权验收、越权修改通知 Key。
- 不把完整资源细节暴露给无权用户；必要时返回“无权限访问”或等价的不可见状态。

### 用户级 IP 白名单

IP 白名单是用户级安全策略，由超级管理员在用户详情中开启和维护。开启后，该用户只允许从指定 IP 或网段登录、刷新会话、访问系统、调用接口、加入文档/表格协同房间、预览或下载文件。

配置能力：

- 开启 / 关闭白名单。
- 新增单个 IP、CIDR 网段和 IPv6 地址。
- 为每条白名单写备注，例如办公室、宿舍、公司 VPN。
- 删除或停用某条白名单。
- 查看最近拦截记录。

校验规则：

- 登录时校验来源 IP，未命中直接拒绝登录。
- 已登录用户请求来源 IP 改变后，每次 API、WebSocket 和文件签名链接生成时重新校验。
- 未命中白名单时销毁或冻结当前会话，避免凭证在非授权网络继续使用。
- 反向代理部署时，只信任配置过的代理来源；真实 IP 只能从可信代理传递的头中读取，禁止直接信任任意 `X-Forwarded-For`。
- IP 白名单变更必须写审计日志，且不向普通用户展示完整白名单。

### 会话空闲超时

登录凭证记录 `last_activity_at`。用户每次有效访问、API 调用、文件操作或协同编辑心跳都会刷新最后活动时间；连续未操作满 24 小时后，会话失效，刷新令牌也同时作废。

处理规则：

- 空闲超时后再次访问返回登录失效状态，并跳转登录框。
- WebSocket 协同连接需要定期心跳；超过 24 小时无有效用户活动时断开连接。
- 文件下载签名链接不能延长登录状态。
- 后台任务、消息发送、导入导出 Worker 不算用户本人操作，不能刷新用户会话。
- 用户退出登录、修改密码、被禁用、IP 白名单变更为不匹配时，也应主动销毁相关会话。

### 角色模板操作

- 内置角色：超级管理员、普通用户、项目拥有者、项目管理员、编辑者、评论者和只读访客不可删除。
- 自定义角色支持新建、复制、查看、编辑和删除。
- 权限范围支持查看矩阵、修改进度可见、文件可见、文件下载和提交物验收。

### 权限默认建议

- 项目拥有者：拥有该项目完整管理权和验收权。
- 项目管理员：可管理任务、成员、资料和验收，但不能越权查看其他项目。
- 编辑者：默认可看成员进度，可编辑授权文件，文件下载按项目设置。
- 评论者：可看授权进度和文件，可评论，不能下载受限提交物。
- 只读访客：只看明确授权内容，默认不能下载成员提交文件，也不能验收。

### 审计

权限变更、邀请、移除成员、分享链接、版本回滚、导出数据、项目归档等操作必须进入审计日志。

### 低配部署监控

2 核 2G 40G 极简安装模式下，管理中心需要突出 CPU、内存、磁盘和 Worker 队列，避免系统在低配环境中因为日志、上传文件或后台任务失控。

### Key 安全

飞书和微信公众号 Key 只能加密保存，前端只显示脱敏状态、最近检测时间和测试发送结果，不显示明文。

Key 的查看、修改、测试发送和删除都必须要求管理权限。测试发送也按当前凭证记录操作者，避免普通成员借接口触发外部消息。

## 6. 个性化

### 入口

个性化入口只保留在登录后的头像菜单用户卡片中。点击头像后先弹出用户卡片，再在卡片下方显示皮肤轮播，不再提供其他独立切换页面或主区皮肤面板。

### 第一版能力

- 书信、情书、风铃木三套预设。
- 自定义背景、强调色和模糊度。
- 选中后全局应用，包括导航、按钮、面板、图表和编辑器。
- 支持查看当前皮肤、修改皮肤、上传/删除自定义背景、重置默认皮肤。

### 后置能力

- 复杂背景识别和全自动 UI 配色。
- 每套皮肤独立图标风格。
