# LightTask v12 文档索引

v12 是在 v11 协作闭环增强版基础上的“界面减负版”。本版本不改变项目协同闭环，不删成员进度、文件提交、定向提醒、权限细分和验收统计，而是减少默认露出的入口，把低频信息收进下拉、标签、折叠摘要和右侧辅助区。

## 当前产品一句话

LightTask v12 是一个轻量级多人项目协同系统，以项目主线、成员进度、资料提交、计划对比、消息提醒和验收复盘为核心，帮助小团队清楚知道“谁在做什么、进度快慢多少、文件交了没有、下一步该提醒谁”。

## v12 核心变化

- 保留 v9 的全局框架、登录框、头像个性化、项目工作台、项目文件、消息同步和管理中心。
- 新增成员级任务进度：一个任务可拆到多个成员，每个成员有自己的计划、实际完成、状态、提交物和说明。
- 新增计划基线、当前计划、实际进度三层时间，用于计算快/慢预期多少天。
- 仪表盘新增“我的进度”视角，展示今日、本周、本月完成情况、预计完成日期和快慢天数。
- 项目工作台新增任务上报动作：完成、延期、阻塞、放弃、休息一下、继续下一任务、上传成果、提醒创建者。
- 项目文件合并在线文档、在线表格、附件和文件收集箱；Word、表格、附件点一下即可在同一工作区打开，并支持新建、上传、重命名、删除、导出。
- 甘特图升级为高级筛选：项目主线、原计划对比、当前进度、指定成员、团队完成节点、只看延期、只看我负责。
- 消息同步升级为定向提醒：全部成员、只提醒负责人、只提醒创建者、自定义成员、不提醒。
- 验收归档升级为验收统计报告：成员计划/实际完成、快慢天数、延期次数、放弃次数、阻塞次数、提交文件数量和验收状态。
- 权限模型落地细分：进度可见、文件可见、文件下载、提交物验收四类权限独立配置。
- 仪表盘精简顶部指标：只保留今日待处理、我的进度偏差、待收集文件和风险项目。
- 甘特筛选减负：默认只露出项目主线、我负责、延期、成员；原计划对比、当前进度和团队节点进入视图设置。
- 项目工作台首屏减负：重点区、任务看板和变更时间线优先；成员进度、文件收集箱和验收报告变为摘要入口。
- 消息同步减负：规则列表默认展示，定向提醒对象只在新建或编辑规则时展开。
- 管理中心减负：用户和权限为主，服务器、Key 和审计以状态摘要呈现。
- 功能树补齐 CRUD：项目、任务、成员进度、验收项、项目文件、提交物、消息规则、通知渠道、角色权限、用户资料和个性化配置均明确增删改查或等价操作。
- 安全防护补齐：所有用户操作都绑定有效凭证和后端权限校验，权限不足时禁止写入、下载、导出、验收、发消息和生成文件签名链接。
- 用户级 IP 白名单：开启后该用户只能从指定 IP 或网段登录、访问、操作、加入协同房间和生成文件链接。
- 24 小时空闲超时：登录状态连续未操作满 24 小时后销毁登录凭证，用户需要重新登录。
- 明确低配部署边界：正式支持 `2 vCPU / 2GB RAM / 40GB SSD` Linux 极简安装，低配约束主要作用于后端、数据库、队列、日志、文件和协同服务；前端资源主要由用户电脑承担，只要求避免首屏和编辑器包臃肿。

## 明确部署环境

第一版按 `2 vCPU / 2GB RAM / 40GB SSD` Linux VPS 可安装运行来约束服务端设计。推荐 Ubuntu 22.04 LTS / 24.04 LTS 或 Debian 12，启用 2-4GB swap 或 zram，使用 Nginx/Caddy 反向代理和 HTTPS，VPS 只部署前端预构建产物和后端服务，不承担前端生产构建、本机高保真 Office 转换、表格转储、全文索引和重型文件处理。前端交互能力不按 VPS 降级，但需要代码分割、懒加载和体积控制，避免普通用户电脑打开时臃肿。

## 第一批已加入核心版

1. 成员级任务进度。
2. 计划基线 / 当前计划 / 实际进度。
3. 个人进度仪表盘。
4. 任务上报动作。
5. 项目文件统一管理：Word、表格、附件、任务提交物和项目文件收集箱。

## 第二批已加入 v12

1. 甘特图高级筛选：默认看项目主线，需要时再展开成员、延期、我负责和团队节点。
2. 定向提醒：任务下发、延期、完成、验收、文件未交等场景可选择提醒对象。
3. 验收统计报告：项目完成后自动汇总成员交付表现和提交物验收状态，用于归档和复盘。
4. 权限细分：把进度可见、文件可见、文件下载和验收权限拆开，避免“能看进度”误等于“能看文件”。

## v12 精简原则

1. 默认界面只回答“现在要做什么、哪里有风险、下一步谁处理”。
2. 高级筛选、完整验收报告、消息对象编辑、服务器细节和审计明细默认折叠。
3. 不删除业务动作；只调整入口层级和首屏占比。
4. 首屏优先级：任务推进 > 风险处理 > 文件提交 > 验收摘要 > 管理和系统信息。
