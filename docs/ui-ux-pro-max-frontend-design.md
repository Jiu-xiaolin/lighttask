# 轻量级多人任务协同系统 - 前端 UI 设计文档

版本：v3 / ui-ux-pro-max 设计重构  
日期：2026-06-01  
定位：面向中小团队的项目排期、任务协同、在线文档、消息提醒与权限管理工作台

## 1. 设计结论

本次重构忽略此前的图片参考和液态玻璃方向，改用更适合协同办公 SaaS 的“Calm Precision”风格：低噪声、信息密集、操作路径清晰、状态反馈明确。页面不做营销式 hero，也不依赖装饰性背景，第一屏直接进入可用工作台。

核心目标：

- 让用户一打开系统就能判断项目是否延期、哪些任务阻塞、谁正在协作。
- 将项目、任务、文档、表格、附件、日志统一进一个项目工作台，减少跨页面跳转。
- 在线文档体验参考石墨文档类协同：多人光标、选区、评论、版本回溯、修改记录。
- 飞书机器人与微信公众号提醒配置要像工作流一样可读、可测试、可追踪。
- 权限管理清楚区分超级管理员、普通用户、项目创建者、项目成员。

## 2. ui-ux-pro-max 适配判断

产品类型：Productivity / SaaS / Dashboard / Admin Panel  
目标用户：项目负责人、研发团队、交付团队、运营管理者、超级管理员  
使用场景：桌面端高频办公，少量移动端查看与处理提醒  
推荐风格：minimalism + structured dashboard + professional SaaS  
不推荐风格：强装饰玻璃拟态、过度渐变、营销首页、低信息密度卡片堆叠

高优先级规则：

- 可访问性：正文对比度不低于 4.5:1，状态不只用颜色表达。
- 交互：按钮/图标可点击区域不小于 44px，异步操作必须有 loading 或成功反馈。
- 布局：桌面端采用固定侧边导航 + 内容工作区；移动端折叠导航，核心内容优先。
- 动效：150-300ms；只使用 transform / opacity；支持 `prefers-reduced-motion`。
- 数据视图：甘特图、时间线、权限矩阵必须有文字标签、图例、筛选和可追溯状态。

## 3. 视觉系统

### 3.1 色彩 Token

| Token | 色值 | 用途 |
|---|---:|---|
| `--bg` | `#F5F7FA` | 页面背景 |
| `--surface` | `#FFFFFF` | 卡片、面板 |
| `--surface-soft` | `#F8FAFC` | 表头、轻分组 |
| `--ink` | `#111827` | 主文本 |
| `--muted` | `#64748B` | 次文本 |
| `--line` | `#E2E8F0` | 边框/分割 |
| `--primary` | `#2563EB` | 主操作、选中 |
| `--teal` | `#0F9F8D` | 正常推进/成功 |
| `--amber` | `#B7791F` | 风险/待确认 |
| `--rose` | `#D8475A` | 阻塞/失败 |
| `--violet` | `#6D5EF7` | 文档协作/版本 |

### 3.2 字体与密度

- 字体族：`Inter, Segoe UI, PingFang SC, Microsoft YaHei, Arial, sans-serif`
- 标题：24 / 20 / 18px，字重 650 或 700。
- 正文：14 / 15px，行高 1.5。
- 数据数字：使用 `tabular-nums`，避免进度、时间和数量跳动。
- 面板圆角：8px，避免过圆导致后台系统显得松散。
- 间距：4/8px 体系；页面区块 24px，组件内 12/16px。

### 3.3 组件原则

- 左侧导航：固定 256px，可折叠为 72px 图标栏。
- 顶部栏：页面标题、范围筛选、搜索、通知和主操作。
- 卡片：只承载一个明确对象，不嵌套卡片。
- 表格：表头固定，行高 48px，支持状态标签和快捷操作。
- 时间线：左侧时间轴，右侧日志卡片；颜色表示类型，文本表示动作。
- 协作文档：顶部工具栏 + 正文编辑区 + 右侧版本/评论面板。
- 权限矩阵：角色为行，能力为列，支持按项目范围覆写。

## 4. 页面设计

### 4.1 仪表盘

信息结构：

- 顶部 KPI：进行中项目、延期风险、本周完成、待处理审批。
- 甘特图：日/月/季度/年分段切换；任务条展示负责人头像、完成度、风险色。
- 项目进度详情：项目卡片列表，每个项目显示里程碑、任务完成率、阻塞项。
- 右侧洞察：今日变更、提醒队列、风险原因。

交互：

- 切换时间维度时甘特时间轴横向平移，任务条使用 180ms 过渡。
- 点击任务条打开任务详情抽屉。
- 风险项目可一键筛选到项目列表。

### 4.2 项目列表 / 工作台

信息结构：

- 项目列表：状态、负责人、进度、最近活动。
- 项目详情：任务看板、成员邀请、附件、在线文档/表格入口。
- 变更时间线：任务、项目、文档、附件、成员变更用不同色条标记。

交互：

- 新建任务使用右侧抽屉，减少页面跳转。
- 邀请成员支持角色选择和项目权限预览。
- 时间线支持按“任务 / 文档 / 附件 / 成员 / 系统”筛选。

### 4.3 在线文档与表格

功能定位：

- 类石墨文档的多人协同体验。
- 文档支持多人光标、评论、版本回溯、修改记录。
- 表格保留常用公式、冻结行列、筛选、格式、导入导出。
- Office / WPS 兼容优先走服务端转换与格式映射，保留样式、批注、基础公式。

交互：

- 顶部显示正在编辑用户，正文出现彩色光标和选区。
- 保存状态展示“已同步 / 正在同步 / 离线草稿”。
- 版本记录以卡片形式同步进入项目日志时间线。
- 导入文件先进入预览确认，显示兼容性检查结果。

### 4.4 消息同步

信息结构：

- 渠道卡片：飞书机器人、微信公众号。
- 触发规则：任务到期、任务阻塞、文档评论、成员加入、项目延期。
- 发送日志：渠道、接收人、状态、失败原因、重试。

交互：

- 配置流程采用三步：连接渠道、选择事件、测试发送。
- 每条规则有开关、静默时段、接收范围。
- 失败日志提供“重试 / 复制错误 / 查看 payload”。

### 4.5 权限管理

角色定义：

- 超级管理员：查看全部数据、管理全局用户、审计日志、系统配置。
- 普通用户：查看自己参与的项目，可创建项目。
- 项目创建者：拥有该项目完全管理权，可邀请成员、配置角色、删除项目。
- 项目成员：按项目角色获得查看、编辑、评论、导出等权限。

界面：

- 权限矩阵直接展示角色和能力。
- 项目级覆写以醒目标签显示，避免误认为是全局权限。
- 审计日志记录权限调整、成员移除、导出敏感文件等操作。

## 5. 动效与反馈

- 页面进入：内容区 `opacity + translateY(8px)`，220ms。
- 面板打开：右侧抽屉 `translateX`，240ms。
- 按钮点击：背景色和阴影 160ms，不改变布局尺寸。
- 列表筛选：行项目 30ms 阶梯淡入。
- 协同光标：位置变化 120ms，颜色稳定绑定用户。
- 支持减少动态效果：启用后移除位移，仅保留必要淡入。

## 6. 响应式策略

- `>=1200px`：完整侧边栏 + 三列工作台。
- `768-1199px`：侧边栏压缩，右侧洞察面板下移。
- `<768px`：顶部导航 + 单列内容；甘特图横向滚动但容器不撑破视口。
- 表格、甘特、权限矩阵等宽内容使用内部滚动，并保留表头/首列。
