# 项目工作台 v5 优化说明

日期：2026-06-01

## 1. 调整目标

本轮只优化项目工作台。核心变化是将“变更时间线”从主内容区域的普通第三列，改为右侧活动侧边栏；主内容区域则围绕项目操作重新组织，减少日志信息对任务处理的干扰。

## 2. 操作逻辑调整

新的项目页按三个区域组织：

- 左侧项目列表：用于切换项目、筛选我负责和风险项目。
- 中间主操作区：展示项目概览、任务/文档/附件/成员/设置视图、任务看板、当前选中任务详情和最近资产入口。
- 右侧变更侧边栏：固定展示项目变更时间线，支持按全部、任务、文档、成员筛选，并提供日志搜索、导出日志和查看审计。

用户路径变为：

1. 从项目列表选择项目。
2. 在主操作区查看项目状态和待处理任务。
3. 选择任务后直接查看详情、提醒负责人、添加评论或标记完成。
4. 需要追溯时查看右侧变更时间线，不打断主操作。

## 3. 显示逻辑调整

项目主区新增：

- 项目概览卡：项目进度、待处理数量、成员数量、资产数量。
- 视图切换：任务、文档、附件、成员、设置。
- 任务看板视图切换：看板、列表、甘特。
- 当前选中任务详情：负责人、截止时间、依赖、完成度和下一步动作。
- 最近文档与最近附件：作为项目资产入口。

右侧活动侧边栏新增：

- 明确的“侧边栏 / 变更时间线”标题层级。
- 类型筛选和搜索框。
- 变更卡片保留前后对比。
- 底部操作：导出日志、查看审计。

## 4. 设计收益

- 时间线仍常驻可见，但不再抢占主操作区的任务处理空间。
- 项目页信息层级更明确：先项目概览，再任务处理，再资产入口，最后变更追溯。
- 用户可以把变更侧边栏当作上下文辅助，而不是每次处理任务都被日志打断。
